Interaction Specifications

AC3 — Soft Delete (5s undo)
  1. User clicks "Delete" in kebab menu
  2. Deck card removed from grid immediately (optimistic UI)
  3. 5s undo toast with countdown progress bar appears
  4. "Undo": deck restored, queued DELETE call cancelled
  5. 5s elapsed: DELETE /api/decks/:id fires → 204
  6. API error after 5s: restore deck to list, show error toast
AC3 — Inline Rename
  1. User clicks "Rename" in kebab menu
  2. Deck name becomes <input> pre-filled
  3. Focus on input, Enter = save, Esc = cancel
  4. Blur = save (click outside)
  5. PATCH /api/decks/:id fires on save
  6. Empty name → inline error "Name required", no API call
AC3 — Change Format (same game only)
  1. User clicks "Change Format" in kebab menu
  2. Inline format panel opens below deck name
  3. Dropdown shows formats for current game only (hardcoded)
  4. Save: PATCH /api/decks/:id → format badge updates in place
  5. Cancel: panel closes, no change
  6. Game field: always locked — not shown in this flow
AC1 — Error States (modal)
  1. Empty name: "Create Deck" button disabled
  2. Whitespace-only name: button disabled
  3. Name at 100 chars: char counter turns red, input rejects further input
  4. API 5xx: inline error in modal footer, button re-enabled
  5. No game selected: "Create Deck" disabled until tile clicked

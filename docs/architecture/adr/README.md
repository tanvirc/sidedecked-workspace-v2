# Architecture Decision Records (ADR)

This directory contains Architecture Decision Records referenced by the architecture documents.

## Purpose

Use ADRs to capture major technical decisions, context, alternatives considered, and consequences.

## Naming Convention

- `ADR-001-<short-title>.md`
- `ADR-002-<short-title>.md`

## Recommended Template

Each ADR should include:
- Title and status (`Proposed`, `Accepted`, `Superseded`)
- Context and problem statement
- Decision
- Alternatives considered
- Consequences and tradeoffs
- Links to affected code/docs/specifications

# Story 6.1: Profile & Activity Feed

## Goal
Deliver rich community profiles and activity feeds.

## Context
Epic 6: Community & Engagement

## Dependencies
- story-01-03-onboarding-profile-management.md

## Acceptance Criteria
1. Display bio, badges, decks, listings, and privacy-controlled sections on public profiles.
2. Aggregate deck updates, purchases, event RSVPs, and forum posts with filtering.
3. Enable follow and unfollow controls that personalize feeds and notifications.

## Implementation Tasks
- Build profile pages with modular sections, privacy controls, and shareable URLs.
- Implement activity feed aggregation service sourcing events across domains.
- Add follow graph storage and preference handling for personalized feeds.

## Validation Plan
- Run UI tests covering profile visibility settings and feed filtering.
- Verify follow relationships update feeds and notification preferences in tests.
